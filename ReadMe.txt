Donor Maintenance System User Guide
-------------------------------------------

Overview

The Donor Management System is designed to manage donor information efficiently. It allows users to create, update, delete, and view donor records, as well as search, filter, and categorize donors. The system is intended for use by organizations managing donors and their contributions.

System Workflow

Main Menu
Upon running the system, the main menu is displayed, allowing you to choose from several options:

1. Create Donor

Step-by-Step Process:

1. Enter Donor Details:
   - Input donor information including Name, Contact Number, Email, Address, Donor Type (Individual or Organization), and Donation Preference.
   - The system validates each input to ensure it meets the required format and criteria.

2. Generate Donor ID:
   - A unique Donor ID is automatically generated by the system.

3. Confirmation:
   - Once all details are validated, the system creates the donor record and displays a confirmation message with the donor details.

Example Workflow:

Enter Donor Name: John Doe
Enter Contact Number: 0123456789
Enter Email: johndoe@example.com
Enter Address: 123 Main Street, Kuala Lumpur
Enter Donor Type (Individual/Organization): Individual
Enter Donation Preference: Cash

2. Update Donor

Step-by-Step Process:

1. Enter Donor ID:
   - Input the Donor ID of the record you wish to update.
   - The ID is validated, and the corresponding donor record is retrieved.

2. Select Field to Update:
   - Choose which field to update (Name, Contact Number, Email, Address, Donor Type, or Donation Preference).
   - The system validates the updated input.

3. Confirmation:
   - After the update, the system displays a confirmation message with the updated donor details.

Example Workflow:

Enter Donor ID to update: D123
Choose field to update:
1. Name
2. Contact Number
3. Email
4. Address
5. Donor Type
6. Donation Preference
Enter your choice: 2
Enter new Contact Number: 0987654321

3. Delete Donor

Step-by-Step Process:

1. Enter Donor ID:
   - Input the Donor ID of the record you wish to delete.
   - The ID is validated, and you are asked to confirm the deletion.

2. Confirm Deletion:
   - Upon confirmation, the system deletes the donor record and displays a message confirming the action.

Example Workflow:

Enter Donor ID to delete: D456
Are you sure you want to delete this donor? (yes/no): yes

4. View Donor

Step-by-Step Process:

1. Enter Donor ID:
   - Input the Donor ID whose details you wish to view.
   - The ID is validated, and the donor details are displayed.

2. View Donations:
   - Optionally, view all donations made by the selected donor.

Example Workflow:

Enter Donor ID to view: D789

5. Search Donor

Step-by-Step Process:

1. Enter Search Criteria:
   - Search donors by ID, Name, Contact Number, or Email.
   - The system retrieves and displays matching donor records.

Example Workflow:

Enter search criteria (ID/Name/Contact/Email): Name
Enter name to search: John Doe

6. Filter Donor

Step-by-Step Process:

1. Choose Filter Type:
   - The system presents filtering options: by donor type, donation preference, or donation history.

2. Provide Filter Criteria:
   - Input the relevant criteria based on your chosen filter type.

3. View Filtered Results:
   - The system filters and displays donors that match the provided criteria.

Example Workflow:

1. Filter by Donor Type
2. Filter by Donation Preference
3. Filter by Donation History
Enter your choice: 1
Enter Donor Type (Individual/Organization): Individual

7. Generate Report

Step-by-Step Process:

1. Select Report Type:
   - Choose the type of report you want to generate (e.g., summary of all donors, categorization by donation type).

2. View Report:
   - The system compiles and displays the report based on the selected criteria.

Example Workflow:

1. Donor Summary Report
2. Donor Categorization Report
0. Return to menu
Enter your choice: 1

8. Exit

Process:
Choose this option to safely exit the system.

Validation and Error Handling

The system includes robust validation to ensure the integrity of input data:

- Donor ID Validation: Ensures that Donor IDs are unique and correctly formatted.
- Contact Number Validation: Ensures contact numbers are valid and of the correct length.
- Email Validation: Ensures email addresses are properly formatted.
- Donor Type and Donation Preference: Validates that inputs match allowed values (Individual/Organization for donor type, and Cash/Item for donation preference).

If any input fails validation, the system prompts the user to retry or exit the operation.

Additional Information

CSV File (donorData.csv): The system references this file to store and validate donor data. Ensure it is up-to-date and correctly formatted.

Data Structures: The system uses data structures like HashMap and LinkedList to manage donors efficiently, ensuring fast retrieval and manipulation of records.

Donation Management System User Guide
-------------------------------------------

Welcome to the Donation Maintenance System! This guide provides detailed instructions on setting up and using the system, covering all features and functionalities.

Overview

The Donation Maintenance System is designed for organizations to manage donation records efficiently. It allows you to create, view, filter, delete donations, and generate comprehensive reports. The system relies on donor information stored in a CSV file (donorData.csv).

System Workflow

Main Menu
Upon running the system, the following options are displayed:

1. Create Donation
2. View Donation
3. Filter Donation
4. Delete Donation
5. Generate Report
6. Exit System

1. Create Donation

Step-by-Step Process:

1. Enter Donor ID:
   - Input the Donor ID.
   - The system validates the ID against the donorData.csv file.
   - If the ID is not found, you are prompted to retry or exit.

2. Enter Donation Amount:
   - Input the donation amount in Malaysia Ringgit (RM).
   - The system ensures the amount is a positive number.

3. Enter Date:
   - Provide the donation date in yyyy-MM-dd format.
   - The system validates the date format.

4. Select Payment Method:
   - Choose between online or cash.
   - The input is normalized and validated.

5. Select Donation Type:
   - Choose from FOOD, DAILY_EXPENSES, or CASH.
   - The input is validated against the available options.

6. Enter Notes:
   - Input any relevant notes.
   - The system ensures that the field is not empty.

7. Confirmation:
   - After validation, the system creates the donation record and displays a confirmation message.

Example Workflow:

Enter Donor ID: D123
Enter Amount (RM): 100.50
Enter Date (yyyy-MM-dd): 2024-08-29
Enter Payment Method (online or cash): online
Enter Donation Type (FOOD, DAILY_EXPENSES, CASH): FOOD
Enter Notes: Donated for community kitchen support.

2. View Donation

Step-by-Step Process:

1. Enter Donor ID:
   - Input the Donor ID whose donations you wish to view.
   - The system validates the ID and retrieves associated donations.

2. Choose Sorting Option:
   - Sort donations by:
     - Date (Ascending/Descending)
     - Amount (Ascending/Descending)
   - Donations are displayed as per your selection.

Example Workflow:

Enter Donor ID to view: D123
Choose sorting option:
1. Sort by Date (Ascending)
2. Sort by Date (Descending)
3. Sort by Amount (Ascending)
4. Sort by Amount (Descending)
Enter your choice: 1

3. Filter Donation

Step-by-Step Process:

1. Choose Filter Type:
   - The system presents filtering options: by donation type, date range, donation amount, or payment method.

2. Provide Filter Criteria:
   - Depending on your choice, input the relevant criteria (e.g., date range, donation amount range).

3. View Filtered Results:
   - The system filters donations based on the provided criteria and displays the results.

Example Workflow:

1. Filter by Donation Type
2. Filter by Date Range
3. Filter by Donation Amount
4. Filter by Payment Method
Enter your choice: 1
Enter Donation Type (FOOD, DAILY_EXPENSES, CASH): FOOD

Step-by-Step Process:

1. Choose Deletion Option:
   - Choose to delete a specific donation by entering its Donation ID or clear all donations.

2. Confirm Deletion:
   - For individual deletion, provide the Donation ID.
   - The system deletes the donation and confirms the action.

Example Workflow:

1. Remove a single donation
2. Clear all donations

Enter your choice: 1

Step-by-Step Process:

1. Generate Report:
   - Select the option to generate a Donation Summary Report.

Example Workflow:

Process:
Choose this option to safely exit the system.

Validation and Error Handling

The system is designed with robust validation to ensure the integrity of input data:

- Donor ID Validation: Checks against the donorData.csv file to ensure the donor ID exists.
- Amount Validation: Ensures donation amounts are positive numbers.
- Date Validation: Ensures dates are in the correct format and are valid calendar dates.
- Payment Method and Donation Type: Inputs are validated to match allowed values (online, cash for payment methods, and FOOD, DAILY_EXPENSES, CASH for donation types).
- Notes Validation: Ensures the notes field is not empty.

If any input fails validation, the system prompts the user to retry or exit the operation.

Additional Information

CSV File (donorData.csv): This file should contain the necessary donor data. The system will reference this file to validate donor IDs during donation creation.

Donor Data File (donorData.csv): Ensure this file is up-to-date and correctly formatted to avoid issues with donor ID validation.

Data Structures: The system uses data structures like LinkedList and HashMap to manage donations efficiently, ensuring fast retrieval and manipulation of records.

Donee Maintenance System
===============================
1. Create Donee
	Step By Step Process :
	  1. Input Donee Details (Name, Address, Phone Number, Email, Donee Type , Organization Name).
	  2. Show registered Donee Details.

	Reminder :
	  - Organization Name only applicable for the Donee that represent an organization.

2. Remove donee
	Step By Step Process : 
	  1. Show the full donee list.
	  2. Input Donee ID to be removed.
	  3. Check if the donee exists or not.
	  4. Show Donee with the input ID has been removed.
	
	Reminder :
	  - If the donee is not found, it will show Donee not found.

3. Update donee details
	Step By Step Process :
	  1. Show the full donee list.
	  2. Input Donee ID to update.
	  3. Showing Update Donee Details Menu.
	  4. Choose which field to update.
	  5. Input the Updated Field.
	  6. Finish Update.

	Reminder :
	  - The Donee ID must be correct otherwise it will show donee not found.
	  - The donee ID cannot be updated since it should be generated by the system.
	  - The Organization only can be updated if the donee represents organization.
	  - The details can be updated many times and only the last input for each field will be updated.

4. Search Donee
	Step By Step Process :
	  1. Input Donee ID to search
	  2. Show Donee is found and Donee Details
	  3. Prompt want to update donee details

	Reminder : 
	  - The donee ID must be in the donee list.
	  - If not found a donee, the prompt update donee details will not display.
	  - The update donee details are similar to 3. Update Donee Details.

5. Applying Aid
	Step By Step Process :
	  1. Show Aid Menu.
	  2. Input the menu choice (1-3) to apply different aid.
	  3. Input Donee ID
	  4. Input amount needed
	  5. Distribution Details are shown.

	Reminder
	  - The Aid Menu also allows the user to check available aid(choice 4).
	  - Donee ID must be correct to distribute.
	  - The distribution will fail if the available amount is not enough for the application.
	  - If the amount distributed is from more than 1 donation, it will show multiple distribution details and the total amount distributed will be the input amount needed.
	  

6. Filter Donee
	Step By Step Process :
	  1. Input the type filter(Organization / Individual)
	  2. Input the Location Filter (Eg : Kuala Lumpur)
	  3. Show the donee list with the filter.

	Reminder 
	  - The filter can leave a blank if the user doesn't want to apply this filter.
	  - If both inputs leave blank, it will show a full donee list.

7. Generate Report
	Step By Step Process :
	  1. Report Menu is shown.
	  2. Select which report to generate.
	  3. Show the selected report.

	Reminder 
	  - There are two different reports that can be generated.
	  - Donee Summary Report is showing the total registered donee, and total for each type of donee registered.
	  - Distribution Summary Report is showing two tables, one is the total distributed amount based on donee, and one is the total distributed amount in the whole system.
	
8. Undo Last Action
	Step By Step Process :
	  1. Select the Undo option.
	  2. The action last performed will be undo.
	
	Reminder 
	  - The undo action is only able to undo the add, remove, and update action.

9. Redo Last Action
	Step By Step Process :
	  1. Select the Redo option.
	  2. The action last undo will be redo.

	Reminder
	  1. The redo action will only redo the undo action.



Volunteer and Event Management System
Overview

The Volunteer and Event Management System is a Java-based application designed to streamline the management of volunteers and events. The system is built to handle various tasks, such as registering volunteers, assigning them to events, and maintaining a record of these activities. This program is particularly useful for organizations that rely heavily on volunteers and need an efficient way to manage them.

Key Features

Volunteer Management:
The VolunteerMaintenance class is responsible for managing all volunteer-related activities. It includes several key functionalities: 
Register New Volunteers: Collect essential details such as name, phone number, email, address, and experience level. Volunteers are assigned a unique ID and stored in VolunteerData.csv.
Delete Volunteers: Easily remove volunteers from the system by their ID. The system ensures data consistency by updating the CSV file upon deletion.
Search Volunteers by ID: Quickly retrieve and display detailed information about any volunteer using their unique ID.
Filter Volunteers by Experience: Display lists of volunteers filtered by their experience level (Experienced/Non-Experienced).
List All Volunteers: View all registered volunteers, complete with their respective details.
The VolunteerMaintenance class utilizes a custom linked list implementation (LinkedList class) to manage the volunteers. This ensures efficient insertion, deletion, and traversal of volunteer records. The class also interacts with file handling utilities to read and write CSV files, ensuring that data is persistently stored and easily retrievable.

Event Management:
The EventMaintenance class is responsible for managing events and the volunteers assigned to them. Key functionalities include:
Assign Volunteers to Events: Assign volunteers to specific events based on their ID and prevent duplicate assignments. The system records event data in Event.csv.
View Volunteers by Event: Retrieve a list of all volunteers assigned to a specific event by event name.
The EventMaintenance class also uses a linked list to manage the volunteers assigned to each event. It interacts with the VolunteerMaintenance class to ensure that only registered volunteers are assigned to events. The class provides mechanisms to handle potential errors, such as assigning volunteers who are already assigned to another event at the same time.

Reporting:

Generate Volunteer Reports: Create summary reports displaying the number of experienced and 
non-experienced volunteers within the system, providing insights into the volunteer pool.

Data Persistence:

CSV File Storage: All volunteer data and event assignments are stored in CSV files for easy access, backup, and editing. Files include VolunteerData.csv for volunteer information and Event.csv for event assignments.
Entities
Volunteer.java: Represents a volunteer, encapsulating attributes like ID, name, phone number, email, address, and experience level.
Event.java: Represents an event, storing event-specific data, including event ID, event name, and the assigned volunteers.
Data Access (DAO)
FileDao.java: This class handles reading and writing data to CSV files, ensuring data persistence.
Control
VolunteerMaintenance.java: Contains the core logic for managing volunteers and events, coordinating operations between the DAO and boundary classes.
User Interface (Boundary)
VolunteerMaintenanceUI.java: Provides a simple console interface for user interactions. Displays menus and options for managing volunteers and events.
Prerequisites
Before running the system, ensure you have the following installed:
Java Development Kit (JDK) 8 or later
Any Java-compatible IDE (e.g., Eclipse, IntelliJ IDEA)
CSV Libraries: The system leverages Javaâ€™s standard I/O libraries for reading and writing CSV files.

How to Run:
Compile and run the VolunteerMaintenance class in your Java IDE.
Follow the menu prompts to perform various operations, such as registering new volunteers, deleting existing volunteers, searching for volunteers, filtering volunteers, listing all volunteers, assigning volunteers to events, and generating summary reports.

How to Use:
Upon launching the application, you will be presented with the main menu that allows you to perform various operations. Below is a quick guide on how to interact with the system:
Main Menu Options
Register a New Volunteer: Enter the volunteer's details (name, phone, email, address, and experience level). A unique volunteer ID is generated, and the data is stored in the CSV file.
Delete a Volunteer by ID: Enter the ID of the volunteer you wish to delete. The system removes the volunteer from the in-memory list and updates the CSV file.
Search Volunteer by ID: Input the volunteer ID, and the system will return the volunteer's details if they are found in the CSV file.
Filter Volunteers by Experience: Choose whether to filter by "Experienced" or "Non-Experienced." The system will display a list of matching volunteers.
List All Volunteers: Shows a full list of all registered volunteers in the system, along with their details.
Assign Volunteer to an Event: Input the volunteer ID and event name to assign the volunteer to a specific event. The assignment is recorded in the Event.csv file.
View Volunteers by Event: Enter the event name, and the system will list all volunteers assigned to that event.
Generate Volunteer Report: Outputs a summary report showing the count of experienced and non-experienced volunteers.
